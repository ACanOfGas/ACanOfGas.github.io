x86 memory consistency内存一致性介绍

x86 memory consistency
发布日期:   2023-09-24

什么是一致性

众所周知，当前各处理器架构都是多核架构，在多核架构下难免都会提到各种一致性的讨论，比如cache一致性，内存一致性等等。或多或少将cache，内存屏障，cpu架构，强序弱序模型的概念都会有一定的提及。最近看了一些相关文档，对这些概念有了新的理解，故想写下这篇文章，记录并复盘一下。如果有不对的地方，欢迎指正。

首先介绍Cache Coherence和Memory Consistency的概念。

  • Cache Coherence指的是确保memory system在并行系统的行为如同cache不存在。

    一个变量由于cache存在，同时会存在多个location。比如同时位于内存中，core 0和core 1的cache line。需要有一定的协议和规范来保证多个location中的cache具有一致性，即该位置的内存相等。

    最终呈现出的结果就像cache不存在一样。

  • Memory Consistency定义的是在并行系统中（比如多核系统中）加载和存储（load/store）不同位置的内存的行为。比如core 0以store a, b的顺序进行存储，对于core 1而言，可能会以什么样的顺序读取到a，b。即使系统中没有cache，一样会存在该memory order的问题，也就是多个core之间不通过cache，直接读写内存，也会有内存序的问题。

Program/Sequential Ordering

一个程序定义了load和store的顺序。

四种类型的memory operation orderings.

W->R: 写X必须在读Y之前；R->R: 读X必须在读Y之前；R->W: 读X必须在写Y之前；W->W: 写X必须在写Y之前。

这是最简单的memory ordering，完全按照程序的顺序决定的。最简单的实现方法如下图就是在内存那里增加仲裁单元，保证多核串行的访问内存。

Relaxing/Weakly Memory Order

宽松的一致性模型允许违反某些顺序。即违反W->R，R->R，R->W，W->W。

宽松的内存序模型有助于隐藏延迟，提高性能，隐藏memory latency，通过overlap内存访问的操作当他们独立时。

如下图所示

通过write-buffer实现这部分优化；也就是写入内容先存入write buffer，不直接对内存生效，此时可以先load后面的B到r1中。

引入了Write Buffer对于内存行为的影响。

Write Buffer引入Performance。

可以看出write buffer的引入提高了W的性能，W的延迟被R overlap覆盖了。

x86的内存序Total Store Ordering（TSO），也就是允许处理器P可以read B在write A被所有处理器看到之前。

处理器可以移动读到写之前。其他处理器不可以返回A的新值，直到A的写入被其他所有处理器观测到。

Total Memory Order

Total Memory Order除了读取可以越过写入的乱序执行，还包括Store Buffers With Forwarding.

也就是由于store buffer的存在，可能造成两次写入在other processor处理器看来写入顺序不一致，故需要增加store buffers with forwarding。即保证W->W的顺序在其他处理器看来是有序的。

架构的文档大家也可以看sdm卷3文档，可以看到实际的HW Memory Model硬件内存模型的范畴大于今天OS的实际允许行为。

x86 TSO的涵盖范围：

包含了cacheable, write-back内存。x86内存的典型场景，除非一些特殊场景，比如异常，时态访问（即不进行cache的内存），自修改code（比如java jvm代码）。

x86汇编示例

Load不会与其他loads重排序，store也不会与其他store重排序。

不会出现r1==1，r2==0的情况。当r1为1时，表示[y]为1。由于M1和M2操作对Processor 1观测是有序的，[y]等于1表示[x]一定为1。

Stores are not reordered with older loads 存储不能重排序到读上面。

r1==1和r2==1不允许，r1等于1，表示[x]为1。由于M2不会重排，所以在M1指令之前[y]都是0。

此时[x]为1时，M4和M3也不会重排，M3已经发生了，此时r2一定为0。

Loads may be reordered with older stores to different locations 不同位置的load和store可以重新排序

由于storebuffer的存在，此时M2和M4都可以重排到M1和M3之前。

故允许r1==0以及r2==0。

相同位置的loads不会排序到store前面的。

Intra-processor forwarding is allowed 处理器转发

允许处理器0在看到[x]之前先看到[y],也就是先看到自己的M1，在看到M4；

而处理器1允许先看到[y]之前先看到[x]，也就是先看到自己的M4，再看到M1；

此时M2和M3的加载或M5和M6的加载不是有序的。

这被解释为存储缓冲区转发，store-buffer forwarding。当一个存储缓冲区被暂时保存在处理器的存储缓冲区中时，它可以满足处理器自身的负载，但对其他处理器的负载不可见（也不能满足）。

Stores are transitively visiable

Intel64内存模型确保传递的可见性，符合因果规律。也就是一个核的写可以保证按照因果关系传递给其他核。

r1等于1，也是M1的操作在M2时候已经可见。随后开始M3。Intel 64可以保证从核2观测M1的操作早于M3。

另外，处理器2的M4和M5加载不会重排序。因此r2等于1，r3也必然等于1。

Total Order on stores to the same location

相同位置上的两个存储必须被其他的处理器以相同的顺序观测到。

也是不会Process2一个观测为先1后2，而Process3一个观测为先2后1

不同位置的store应该也是有着一样的顺序

Locked Instruction have a total order

Locked instructions加锁的原子指令也是拥有着total order。

Loads and stores are not reordered with locks

M2不可绕过M1，M4不可绕过M3。

R2==0表示M3未发生，故M3时M1已发生，此时r4为1。

总结

以极客时间上的专栏为例，解释了x86内存屏障的意义。
